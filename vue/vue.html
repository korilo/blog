                  # 1. Vue初识

                  • 概述

                  Vue是一套用于构建用户界面的渐进式框架。Vue 的核心库只关注视图层，结合了HTML + CSS + JavaScript，易用。体积小，速度快，优化到位，有很好的生态系统 。

                  • 特点和高级功能：
                    • 轻量级
                    • 解耦视图和数据
                    • 可复用的组件
                    • 前端路由技术
                    • 状态管理
                    • 虚拟DOM

                  # 1.1 MVVM模式

                  Vue技术是MVVM开发模式的实现者

                  • MVVM模式的实现者--双向数据绑定模式
                    • Model：模型层（数据），表示JavaScript对象
                    • View：视图层（视图），表示DOM（HTML操作的元素）
                    • ViewModel：连接视图和数据的中间件，Vue就是MVVM中ViewModel层的实现者

                  • VM实现原理

                    • 在MVVM架构中，是不允许 数据 和 视图 直接通信的，只能通过ViewModel来通信
                    • ViewModel就是定义了一个 Observer观察者
                      • 能够监听视图的变化，并通知数据发生改变**（监听）**
                      • 能够观察数据的变化，并对视图对应内容进行更新**（绑定）**
                    • MVVM通过VM实现了双向数据绑定，Vue的核心就是实现了监听和绑定

                  • 其他MVVM的实现者

                    • Angular
                    • React
                    • 微信小程序

                  # 1.2 虚拟DOM和diff算法

                  Vue是封装了的JavaScript代码，而JavaScript的作用和目的就是改变页面元素的内容和样式，Vue使得这一目的更好的被实现。

                  • JavaScript修改元素的方式
                    • 通过document.get...的方式获取元素
                    • 然后修改DOM树上的节点
                    • 效率低
                  • Vue虚拟DOM
                    • 并非直接修改DOM树上的结点
                    • 模拟出一个虚拟的DOM树，相当于真实DOM的蓝图
                    • 通过diff算法，比对虚拟DOM与真实DOM的区别来修改真实DOM

                  # 2. Vue输出和指令

                  # 2.1 插值表达式 mustache

                  • 作用：
                    • 在View中（动态）获得Model中的内容
                  • 可获取的内容
                    • Vue对象中的属性：data对象和computed对象中的属性
                    • Vue对象中的方法：methods属性中的方法
                    • 差值表达式还支持JavaScript的运算符和语法
                  • 应用结合：
                    • 与v-for遍历时显示遍历结果
                    • 与|过滤器结合使用，对数据进行处理后显示
                  • 注意：
                    • 插值表达式不能写在标签的属性中
                    • 当网速过慢时，会显示插值表达式，有闪烁的现象（解决：v-cloak）

                  # 2.2 v-text

                  • 作用：
                    • 应用：
                      • 为p、span等文本标签里加上文本内容
                    • 实际作用
                      • 更新元素的内容区
                      • 与插值表达式作用类似，但不能更新内容区的部分内容，只能完全替换
                  • 注意：
                    • v-text会覆盖元素中的文本内容，如果原来标签内有内容将会被替换
                    • 要更新部分内容请使用插值表达式
                    • 替换内容时也有闪烁问题，若原来的内容区没有内容，则不会闪烁
                    • 与mustache一样，内容里的html元素将不会被解析，而是作为文本输出
                  <span v-text="msg"></span>
<!-- 和下面的一样 -->
<span>{{msg}}</span>

                  # 2.3 v-html

                  • 作用：
                  • 应用：
                    • 为可嵌套的父级元素里加上解析成html文本的内容
                    • 实际作用
                      • 与v-text对应，但会解析内容里的html元素
                    • 注意：
                      • 插入的html内容不会作为Vue模板进行编译
                      • 要插入组件还是要通过正常插入的方式
                      • 在网站上动态渲染html元素还有一定的安全问题
                      • 动态渲染的html元素无法应用该模组内使用了 scope 的css样式，要解决可以使用CSS Modules 或利用全局style标签设置CSS BEM
                  //vue实例
data(){
	return: {
		web: '<a href="https://www.bilibili.com">哔哩哔哩</a>'
	}
}

<!--html-->
<p v-html="web"></p>

                  # 2.4 v-once

                  • 作用：
                    • 只获取一次model的数据
                    • 后续改变对应的数据时，显示不会跟随改变
                  • 应用：
                    • 一次性展示的场合，不希望数据变化
                    • 取消响应式数据绑定
                  • 注意：
                    • 后面没有表达式

                  # 2.5 v-cloak

                  • 作用：

                    • 应用：
                      • 用于隐藏插值表达式，取消 ''闪烁'' 现象
                    • 实际作用：
                      • 这个指令会保持在元素上直到关联实例结束编译
                      • 即将 v-cloak 作为自定义属性保存在元素上直到第一次取到数据之前（vue解析完成）
                      • 结合css的属性选择器可以使该元素在这个阶段获得自定义的样式

                  • 用法：

                    • 与css属性选择器"[ ]"结合使用，设置样式 display 为 none

                  • 闪烁现象

                    • 在网速较慢的情况下，插值表达式的内容还没加载出来时，会显示原来双括号的形式
                    • vue 2.6之后不会显示花括号，只会等待加载完成后直接渲染，也就不会有闪烁现象了

                  • 语法：

                  <p v-cloak> {{data}} </p>

<style>
    [v-cloak]{
        display: none;
    }
</style>

                  # 2.6 v-bind

                  利用插值表达式可以动态绑定标签的内容，但不能动态绑定标签的属性。插值表达式不能写在属性里，但可以使用v-bind指令来解决该问题

                  • 作用：

                    • 为元素绑定属性
                    <!--将链接地址绑定为web，web是vue对象data里的属性-->
<a v-bind:href="web">超链接</a>

                  • 应用：

                    • 绑定样式
                    <!--将链接地址绑定为web，web是vue对象data里的属性-->
<a v-bind:href="web">超链接</a>

                  • 语法：

                    • v-bind:标签属性="data属性"
                    • 简写：:标签属性="data属性"

                  # 2.5 v-on

                  • 作用：

                    • 绑定事件
                    • 简写@
                    <button v-on:click="method">按钮</button>
<button @click="method">按钮</button>

                  • 语法：

                    • v-on:事件名（不要on）="method(传参)"
                    • 简写：@
                    • 可以直接实现数据的运算：@click=“a++”,data里的a会在每次点击后自增

                  • 注意：

                    • 事件监听中，参数传递相关：
                      • 参数不加引号时，默认是变量，传递字符串时要加引号
                      • 方法不需要参数时，可以省略括号
                      • 方法需要参数但没有参数传递时，可以省略括号
                        • 省略括号时：默认会将event事件传入
                          • 多参数时，会将第一个参数传递event，其他传递undefined
                        • 没省略括号：默认传递undefined
                      • 传递event对象方法：
                        • 当需要传递event对象时，需要加美元符号$event
                        • 不加符号时默认event是个data变量

                  • 事件分类：

                    • 鼠标事件：

                      • 事件名
                        • click
                        • dblclick
                      • 修饰符
                        • .left：鼠标左键
                        • .right：鼠标右键
                        • .middle：鼠标中键
                        • .exact：没有按键盘时点击
                        • .exact.键盘事件修饰符：按键精确的同时点击
                        • .键盘事件修饰符：只要有按键盘时点击，可以是任意键盘修饰符

                    • 滚轮事件：

                      • 事件名

                        • mousewheel

                    • 键盘事件：

                      • 事件名
                        • keyup：键盘抬起
                        • keydown：键盘按下
                      • 修饰符
                        • .ctrl：ctrl键按下
                        • .alt：alt键按下
                        • .shift：shift键按下
                        • .meta：win键按下
                        • .space：空格键按下
                        • .数字：对应的按键unicode
                        • 可多选组合
                        • 自定义键盘修饰符：
                          • Vue.config.keyCodes.按键 = 数字;

                  • 事件修饰符：

                    • 用法：v-on:click.stop=""在事件名后修饰

                    • 可串联，可只有修饰符

                      • 串联顺序问题： 相应的代码会以同样的顺序产生。因此，用 v-on:click.prevent.self 会阻止所有的点击，而 v-on:click.self.prevent 只会阻止对元素自身的点击。
                      • 只有修饰符：v-on:submit.prevent

                    • 分类：

                      • .stop

                        • 阻止事件冒泡
                        <!-- 阻止单击事件继续传播 -->
<a v-on:click.stop="doThis"></a>

                      • .prevent

                        • 取消标签的默认事件，只执行绑定的事件
                        <!-- 提交事件不再重载页面 -->
<form v-on:submit.prevent="onSubmit"></form>

                      • .capture

                        • 触发在捕获阶段
                        <!-- 添加事件监听器时使用事件捕获模式 -->
<!-- 即内部元素触发的事件先在此处理，然后才交由内部元素进行处理 -->
<div v-on:click.capture="doThis">...</div>

                      • .self

                        • 只有本身事件触发才响应
                        <!-- 只当在 event.target 是当前元素自身时触发处理函数 -->
<!-- 即事件不是从内部元素触发的 -->
<div v-on:click.self="doThat">...</div>

                      • .once

                        • 只会触发一次
                        <!-- 点击事件将只会触发一次 -->
<a v-on:click.once="doThis"></a>

                      • .passive

                        • 标签的默认行为将立即触发，不用等事件触发
                        • 不能与.prevent一起用
                        • 可以提高移动端的性能
                        <!-- 滚动事件的默认行为 (即滚动行为) 将会立即触发 -->
<!-- 而不会等待 `onScroll` 完成  -->
<!-- 这其中包含 `event.preventDefault()` 的情况 -->
<div v-on:scroll.passive="onScroll">...</div>

                  # 2.6 v-model

                  • 作用：
                    • 实现双向数据绑定
                    • 将输入的内容（select选择的内容）与data里的属性进行绑定，双向修改
                    • 输出使用插值表达式
                  <input type="text" v-model="data" />

                  • 限制：只有能输入的元素才能使用

                    • <input />
                    • <select />
                    • <textarea />
                    • components（可输入）

                  • 修饰符

                    • .lazy：
                      • 懒加载，取代<input />的change监听，不会实时监听
                      • 鼠标焦点离开时或回车时才触发
                    • .number：
                      • 输入的字符串转换为有效的数字
                      • v-model输入的值默认为string类型
                    • .trim：
                      • 输入的字符串过滤首位空格

                  • v-model结合input标签

                    • text：
                      • 双向绑定value值
                    • radio：
                      • 双向绑定value，data变量记录选定radio的value值
                      • 可以省略name属性也实现互斥单选
                    • checkbox：
                      • 单选框：
                        • 双向绑定checked，data变量记录布尔值
                        • 应用：同意用户协议
                      • 多选框：
                        • 双向绑定value值，data变量记录选定value的数组

                  • v-model结合select标签

                    • 单选
                      • 双向绑定option的value值，data变量记录option的value值
                    • 多选
                      • 双向绑定option的value值，data变量记录选中option的value数组

                  • 值绑定动态生成多选框

                  data: {
	color: [red,green,yellow,blue]
}

<label v-for="item in color" :for="item">
    <input type="checkbox" :value="item" :id="item" v-model="like" />{{item}}
</label>

                  # 2.7 v-if、v-else、v-else-if

                  • 语法：
                    • v-if="data属性（布尔值）"
                    • v-else
                    • v-else-if="data属性（布尔值）"
                  • 作用：
                    • 用来控制页面元素的 创建 和 删除
                    • 修改的是DOM树的内容
                  • 判断逻辑：
                    • v-if为真，后面的都不执行
                    • v-if为假，
                      • v-if-else为真，后面的都不执行
                      • v-else执行
                      • v-if-else为假
                        • v-else执行
                  <!--当 a 为 true 时显示-->
<div class="box1" v-if="a"></div>

<!--当 a 为 false 且 b 为 true 时显示-->
<div class="box2" v-else-if="b"></div>

<!--当 a 为 false 且 b 为 false 时显示-->
<div class="box3" v-else="a"</div>

                  # 2.8 v-show

                  • 语法：v-show="data属性（布尔值）"
                  • 作用：
                    • 控制页面元素的显示和隐藏
                    • 修改的是元素的display属性，切换 visible 和 none
                    • 会空出位置
                  • 适用：
                    • 频繁需要隐藏和显示的元素
                    • 比v-if性能好

                  # 2.9 v-for

                  • 作用：遍历对象、数组

                  • 遍历数组：

                    • 格式：v-for="(value,index) in data属性"
                    • value和index是形参（顺序不可变，形参名可变）
                    • 分别表示其遍历数组的 值、索引
                    • 利用插值表达式显示对应的值

                  • 遍历对象：

                    • 格式：v-for="(value,key,index) in data属性"
                    • value、key和index是形参（顺序不可变，形参名可变），分别表示其遍历对象的 值、键、索引
                    • 利用插值表达式显示对应的值

                  • 遍历对象数组或多对象对象

                    • 方法：
                      • 1）先遍历出每个对象，再调用对象的属性
                      • 2）双重v-for遍历

                  • 注意：

                    • 在li或div标签进行遍历，会将结构一起输出
                    • 可以使用template标签来进行遍历，这样只会输出template标签内的结构
                    <template v-for="(value,index) in arr">
	<p>{{value}}</p>
</template>

                  # 3.特殊 attribute

                  # 3.1 ref

                  • 作用：

                    • 取代元素的id选择器，相当于id
                    • 可以通过访问父类refs来获取到绑定元素

                  • 父类：refs

                    • 相当于一个对象，保存在vue实例的data里
                    • 保存了所有被ref绑定的元素
                    • 可以通过获取对象属性的方法来操作元素
                    • 调用要加美元符$refs
                    <!--另一种方式的双向绑定-->
<input type="text" ref="name" @keyup="changeName" />

data(){
	return {
		name: ''
	}
},
methods: {
	changeName: function(){
		this.name = this.$refs.name.value;
	}
}

                  # 2.3 小技巧

                  • event.target

                    • 在事件绑定函数里可以获取触发事件的元素
                    • 再通过其属性可以访问其特定的属性值

                  • this

                    • 在原生Vue中可以指向该Vue对象
                    • 也通过其属性来调用该Vue对象的属性和方法
                    • 在ajax请求中不能使用this

                  • key属性

                    • key 的特殊属性主要用在 Vue 的虚拟 DOM 算法，在新旧 nodes 对比时辨识 VNodes。如果不使用 key，Vue 会使用一种最大限度减少动态元素并且尽可能的尝试就地修改/复用相同类型元素的算法（元素的复用，性能优化）。而使用 key 时，它会基于 key 的变化重新排列元素顺序，并且会移除 key 不存在的元素。
                    • 有相同父元素的子元素必须有独特的 key。重复的 key 会造成渲染错误。

                  • 最常见的用例是结合 v-for：

                    • 目的：
                      • 更好的复用，以达到性能优化
                      • 在虚拟dom中会让元素和绑定的key值进行对比，若没有变化则不会重新渲染
                    • 注意：
                      • 一般要绑定一个唯一对应的值，如value
                      • 不要使用index，当在中间进行修改时，元素对应的index就会改变
                    <ul>
  <li v-for="item in items" :key="item.id">...</li>
</ul>

                  • 它也可以用于强制替换元素/组件而不是重复使用它。当你遇到如下场景时它可能会很有用：

                    • 完整地触发组件的生命周期钩子
                    • 触发过渡

                    例如：

                    <transition>
  <span :key="text">{{ text }}</span>
</transition>

                    当 text 发生改变时，`` 总是会被替换而不是被修改，因此会触发过渡。

                  # 3.3 生命周期钩子

                  生命周期（钩子）函数，实际上是vue实例对象在执行过程中调用的回调函数，在不同时间结点会被进行调用

                  • beforeCreate
                    • 在实例初始化之后，数据观测 (data observer) 和 event/watcher 事件配置之前被调用。
                  • created
                    • 在实例创建完成后被立即调用。在这一步，实例已完成以下的配置：数据观测 (data observer)，属性和方法的运算，watch/event 事件回调。然而，挂载阶段还没开始，$el 属性目前尚不可用。
                  • beforeMount

                    • 在挂载开始之前被调用：相关的 render 函数首次被调用。

                    • 该钩子在服务器端渲染期间不被调用。

                  • mounted

                    • 实例被挂载后调用，这时 el 被新创建的 vm.$el 替换了。如果根实例挂载到了一个文档内的元素上，当 mounted 被调用时 vm.$el 也在文档内。

                    • 注意 mounted 不会保证所有的子组件也都一起被挂载。如果你希望等到整个视图都渲染完毕，可以在 mounted 内部使用 vm.$nextTick：

                    mounted: function () {
  this.$nextTick(function () {
    // Code that will run only after the
    // entire view has been rendered
  })
}
                    • 该钩子在服务器端渲染期间不被调用。
                  • beforeUpdate

                    • 数据更新时调用，发生在虚拟 DOM 打补丁之前。这里适合在更新之前访问现有的 DOM，比如手动移除已添加的事件监听器。

                    • 该钩子在服务器端渲染期间不被调用，因为只有初次渲染会在服务端进行。

                  • updated

                    • 由于数据更改导致的虚拟 DOM 重新渲染和打补丁，在这之后会调用该钩子。

                    • 当这个钩子被调用时，组件 DOM 已经更新，所以你现在可以执行依赖于 DOM 的操作。然而在大多数情况下，你应该避免在此期间更改状态。如果要相应状态改变，通常最好使用计算属性或 watcher 取而代之。

                    • 注意 updated 不会保证所有的子组件也都一起被重绘。如果你希望等到整个视图都重绘完毕，可以在 updated 里使用 vm.$nextTick：

                    updated: function () {
  this.$nextTick(function () {
    // Code that will run only after the
    // entire view has been re-rendered
  })
}
                    • 该钩子在服务器端渲染期间不被调用。
                  • activated
                    • 被 keep-alive 缓存的组件激活时调用。
                    • 该钩子在服务器端渲染期间不被调用。
                  • deactivated
                    • 被 keep-alive 缓存的组件停用时调用。
                    • 该钩子在服务器端渲染期间不被调用。
                  • beforeDestroy

                    • 实例销毁之前调用。在这一步，实例仍然完全可用。

                    • 该钩子在服务器端渲染期间不被调用。

                  • destroyed
                    • 实例销毁后调用。该钩子被调用后，对应 Vue 实例的所有指令都被解绑，所有的事件监听器被移除，所有的子实例也都被销毁。
                    • 该钩子在服务器端渲染期间不被调用。

                  # 5. 动态绑定样式

                  JavaScript中绑定样式有两种方式，一是修改元素的类名，来获取类名所对应的样式。另一种就是修改元素的style属性，通过写入行内样式来修改元素的样式。vue要实现动态绑定样式，也可以从这两个方法入手。

                  # 5.1 动态绑定class

                  • v-bind绑定class的特点：

                    • 动态绑定的多个class会进行合并
                    • 传值可以是字符串、数组和对象

                  • 使用方式：

                    • 对于绑定单个确定的样式类名，可以直接传值类名
                    • 绑定多样式时，有两种传值方式：
                      • 传递对象
                      • 传递数组
                    • 可以根据需求的不同，选择合适的传值方式
                  • 对象语法

                    • 特点：

                      • 键表示类名
                      • 值是Boolean值，决定是否挂载该样式
                      <div v-bind:class="{red: isred}" ></div>

                    • 应用：

                      • 根据存储的boolean动态控制样式显示
                      • 可以使用methods或computed方式传回样式对象

                    • 注意：

                      • 对象的键在解析时默认是字符串，是不会到data对象里找值的
                        • 不可用通过修改变量的方式来改变类名
                      • 要动态修改样式名可以通过computed或methods方式
                        • 用this.属性名来动态传递类名（也没什么必要，很少使用）

                  • 数组语法

                    • 特点：

                      • 每个数组元素控制一个类名
                      • 数组元素可以是字符串或对象
                      <div v-bind:class="['样式名1',{样式名2： boolean}]" ></div>

                    • 应用：

                      • 当类非常多时，可以将类名放在数组里
                      • 可以使用methods或computed方式传回样式对象
                      • 与对象嵌套，实现部分样式动态绑定

                    • 注意：

                      • 与绑定单样式不同，数组元素传递的类名必须要加引号
                      • 不加引号的话vue会以为是data属性，会在data对象里找，没有找到就会报错
                  • 绑定computed属性
                    • 动态绑定样式类名的最优方法
                    • 作用：
                      • 在computed里声明一个方法，返回一个jason对象或数组
                      • 对象或数组里包含样式和布尔值的键值对，可以用于多个样式的整合
                    • 优点：
                      • 可以解决对象类名的动态传值
                      • 可以更方便的管理样式
                  <div v-bind:class="myStyle" ></div>

computed:{
	myStyle: function(){
		return{
			this.style : this.isred,
			
		}
	}
}
