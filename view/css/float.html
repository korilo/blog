# 块盒浮动

浮动的盒子都是块盒

如果元素原来不是块盒，将会强制转换成块盒

强制转换：display: block


# 浮动元素包含块

浮动元素的包含块和文档流的一样，也为父元素的内容盒content


# 盒子auto含义

# width: auto

文档流：块级元素的width默认值是auto(填充剩余空间)

浮动后：width的默认值虽然还是auto，但却变成内容撑开，与文档流里height的auto意义一样


# height: auto

浮动后：height的auto与文档流里一致，由内容撑开


# margin: auto

文档流中：水平方向margin设置为auto时会填充剩余空间，优先级低于width

浮动后：水平方向margin设置为auto时与设置成0等效


# 浮动盒子位置规则

  1. 左浮动的盒子向上向左排列
  2. 右浮动的盒子向上向右排列
  3. 浮动盒子的顶边不得高于上一个盒子顶边
  4. 若剩余空间无法放下浮动的盒子，则该盒子向下移动，直到具备足够的空间能容纳盒子，再根据左（右）浮动向左向右移动

注意：当浮动盒子的width没有设置或为auto时，盒子的宽度由内容撑开，当剩余空间小于撑开后的宽度时，盒子向下移动，在新的一行显示。若撑开后的宽度还是大于页面宽度，浮动盒子的内容才进行换行


# 匿名行盒

  • 浮动盒子的排列规则：
    • 浮动盒子不会超过前一个文档流块盒
    • 行盒在排列时会避开浮动盒子

利用第二个规则，可以实现文字环绕图片的效果

提问：p标签是块盒，为什么文字会避开浮动图片

原因：浏览器会自动在文字外生成一个匿名行盒来保存文字内容，p标签是块盒，实际上并没有避开浮动元素，而是被覆盖，而文字内容被匿名行盒包裹，最终产生了只有文字避开了浮动元素的效果

# 高度塌陷

高度塌陷的根源：

文档流盒子的自动高度，在计算时，不会考虑浮动盒子内容

解决方法：

  1. 清除浮动

注意：清除浮动不是说不浮动，是让块盒不无视浮动盒子。注意清除浮动的对象

  • clear属性
    • left: 清除左浮动，该元素必须出现在前面所有左浮动盒子的下方
    • right: 清除右浮动，该元素必须出现在前面所有右浮动盒子的下方
    • both: 清除所有浮动，该元素必须考虑浮动盒子的位置

能够解决高度塌陷的原因：

文档流中，在父盒子的末尾插入一个空盒子，对其清除浮动，该盒子就会考虑浮动盒子的位置，从而显示在浮动盒子下方，而父盒子计算到该空盒子的位置，就撑开了父盒子

一般利用css伪元素来实现相同的功能：

.clearfix::after{
    content: "";
    display: block;
    clear: both;
}

注意：该类应该给父容器

  1. 开启BFC

设置overflow: hidden
