# 视觉格式化模型（布局规则）

大体上将页面中盒子的排列分成三种方式

  1. 文档流（常规流）
  2. 浮动
  3. 定位

# 包含块

包含块：containing block

每个盒子都有它的包含块，包含块决定了盒子的活动范围（排列区域）

绝大部分情况下，盒子的包含块为父元素的内容盒content

# 文档流

默认情况下，页面中的所有元素都是常规流布局

特点：

  1. 块盒独占一行，行盒水平依次排列
  2. 自上而下布局

# 块盒特点

# 独占一行

每个块盒的总宽度，必须刚好等于包含块的宽度（独占一行）

独占一行的原因：

  • 行相关的默认值
    • width: auto(填充剩余空间)
    • margin: 0
    • padding: 0
    • border: 0

默认width会吸收包含块行内所有的剩余空间

当设置上面多个属性值为auto时，width的优先级最高

当margin、padding、border、width都设置完后，如果还有还有剩余空间，将计算进margin-right

当margin-left和margin-right同时设置为auto，将达到水平居中的效果（margin-left/marrginright的优先级相等）

总结：

浏览器为了保证块盒独占一行，使用了以下规则

  1. width默认值为auto，保证了什么都不做的情况下能独占一行
  2. 自动分配剩余空间给margin-right，保证了块盒宽度固定时能独占一行

# 常规流中块盒水平居中方式（拓展）

设置margin-left和margin-right为auto

左右外边距会吸收掉行内的剩余空间，达到居中的效果

注意：width的优先级比margin高，若width没有赋值或赋值auto，内容区将占满整行

# 垂直方向默认值

  • 竖直方向相关默认值：
    • height: auto(内容撑开)
    • margin: auto(0)

# %百分比参考

特点：

margin、padding无论是水平方向还是竖直方向上，参考的都是包含块的宽度

高度百分比的特点：

  • 当父元素设置了确定的高度时，百分比才有效

当包含块的高度没有设置，即由子元素撑开时，百分比无效

当包含块的高度确定时，百分比相对于包含块高度

# 垂直方向相邻外边距重叠

  • 兄弟元素不同边外边距相邻时会发生重叠现象，取最大值进行显示

  • 父子元素同边外边距相邻时会发生重叠现象，取最大值进行显示，造成子元素无法与父元素边框间隔距离

造成外边距重叠的条件是相邻，只要打破这个条件，就不会产生重叠现象

  • 打破相邻的常用方法：
    • 设置边框
    • 用padding代替margin
    • 用伪元素插入块级空内容隔开

注意：只有文档流块盒才有外边距重叠，浮动、定位都没有
