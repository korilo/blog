# 盒子的组成

无论是行盒还是块盒，都由下面几个部分组成

  1. content
  2. padding
  3. border
  4. margin

除了content，其他都由top/bottom/left/right 4个部分组成，对应padding-top、padding-bottom等

  • width、height默认设置为内容盒content-box

默认情况下，width 和 height 设置的是content的宽高，这样会造成一个状况： 当设置padding、border时，盒子整体会被撑开，造成样式与设计出现差别

解决：将原来的content-box 设置成border-box

设置box-sizing: border-box （css3）

设置后 width 和 height 设置的是以边框为基准的宽高

  • 背景默认设置是边框盒border-box

背景默认显示区域是边框盒，包含边框

改变显示区域：修改属性background-clip

可以修改为content-box、padding-box、border-box

# 块盒

块盒：display: block

常见块盒：容器元素div、header等、h、p

特点：

  1. 独占一行
  2. 可以设置宽高

# 行盒

行盒：display: inline

常见的行盒：包含具体内容的元素

span、strong、em、i、img、video、audio

特点：

  1. 盒子沿着内容延伸
  2. 设置宽高无效，可以用字体大 小、行高等属性间接调整宽高
  3. padding、margin、border:水平方向有效，但竖直方向只能看到效果，实际内容没有动

# 行块盒

行块盒: display: inline-block

特点：

  1. 在行内显示，也可以设置宽高
  2. 行盒和行块盒的内容间会发生空白折叠，造成盒子之间有间隙（可以不空行写，就不会产生间隙）

应用场景：分页

# 可替换元素

可替换元素：显示内容取决于属性的元素，如img/audio/video

非可替换元素：显示内容取决于元素内容的元素

页面上绝大部分元素都是非可替换元素

绝大部分可替换元素均为行盒，display: inline

但是，可替换元素却拥有行块盒的特性而非行盒的特性，可以设置宽高

原因：

可替换元素有一个隐藏属性：object-fit

  • 属性值：
    • fill: 默认值，填充显示，拉伸宽高
    • cover: 保持比例填充，溢出部分隐藏
    • contain：保持比例，完整显示
