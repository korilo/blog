# 计算过程

浏览器渲染html元素是一个元素一个元素按次序渲染的，顺序按照页面文档的树形目录结构进行

渲染每个元素的前提条件：该元素的所有css属性必须都有值（所有属性都被赋值）

一个元素，从所有属性都没有值，到所有属性都被赋值的过程，叫做属性值的计算过程

css元素属性值计算过程分为4个步骤：

  1. 确定声明值
  2. 层叠冲突
  3. 使用继承
  4. 使用默认值

经过4个步骤后，该元素的的所有属性都被赋予了值，浏览器开始渲染元素

# 1.确定声明值

该过程会参考样式表中没有冲突的声明，作为css属性值

参考的样式表包括自己写的声明样式表以及浏览器的默认样式表

注意：浏览器对部分元素有默认的样式声明，要注意区分样式默认值和浏览器默认样式，这是两个不同的概念

一些属性的简写写法（语法糖），实际上会转换成对每个具体的属性赋值，保证每个属性都被赋值

# 2.层叠冲突

该过程会对样式表有冲突的声明使用层叠规则，确定css属性值

  • 比较重要性（非默认）
  • 比较特殊性（选择器权重）
  • 比较源次序（声明先后）

经过上面三个步骤确定冲突属性的最终值

# 3.使用继承

该过程会查看没有被赋值的其他属性是否有继承属性

若可继承，则继承上级元素的属性值

# 4.使用默认值

若还是有属性没有被赋值，将全部按照默认值进行赋值

至此，该元素的所有属性都被赋值，浏览器开始对元素进行渲染

# 易踩的坑

说出a标签的颜色

<div class="wrap" style="color: red">
    <a>123</a>
</div>

结果是： a标签的文字颜色还是蓝色，并没有继承到父级的属性值

原因：a标签有默认样式，指定了字体的颜色，所以不会发生第三步的继承

若有这样一个需求：要求a标签能继承父级元素的字体颜色样式，要怎么实现？

首先：a标签字体颜色有默认样式前面已经说了，无法正常继承父级样式

其次：若自己声明了a标签的字体颜色，则会在第二步中替换掉默认样式，同样无法继承父级样式

解决方法：声明字体颜色样式属性值为inherit，强制继承父级元素样式

  • 类似的特殊属性值：
    • inherit: 继承父元素属性值
    • initial: 默认值
