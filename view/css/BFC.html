# BFC布局规则

BFC: Block Formatting Context

BFC是一块独立的 渲染区域，规定了在该区域 内部，文档流块盒的布局

BFC内部元素渲染规则：

  1. 文档流块盒在水平方向上必须撑满包含块
  2. 文档流块盒在包含块的垂直方向上自上而下布局
  3. 文档流块盒相邻外边距合并
  4. 文档流块盒的自动高度和摆放位置无视浮动元素

# BFC渲染区域

BFC渲染区域由某个HTML元素在其 内部 创建

能创建BFC的元素：

  1. 根元素，即html元素生成一个包含网页中所有元素的BFC区域
  2. 浮动和绝对定位元素
  3. overflow不等于visible的块盒

# BFC渲染规则

不同的BFC区域，它们进行渲染时互不干扰

创建BFC的元素 隔绝了它内部和外部的联系，内部的渲染不会影响到外部

开启BFC的元素的渲染规则：

  1. 创建BFC的元素，它的自动高度需要计算 浮动 元素，但不计算完全脱离文档流的绝对定位和固定定位元素
  2. 创建BFC的元素，它的边框盒不会与浮动元素重叠
  3. 创建BFC的元素，不会和它的子元素进行外边距合并（不同BFC渲染区域内的元素的布局互不影响）

以上规则都是为了使内部元素与外部元素渲染隔绝

# 高度塌陷

对父盒子开启BFC可以解决文档流块盒的高度塌陷、外边距重合的问题

要开启某元素的BFC，使用浮动和绝对定位是不合理的，会影响元素本身的布局

副作用最小的做法是设置overflow属性为hidden

解决高度塌陷方法总结：

  1. 文档流中，父盒子插入清除浮动的空盒
  2. 文档流中，父盒子开启BFC

# 多栏布局

利用开启BFC元素的特性，可以使文档流块盒避开浮动盒子进行渲染，达到多栏布局的效果

详细：

  1. 左（右）栏开启浮动
  2. 适应栏开启BFC

为了防止适应栏缩小太多影响观感，一般会设置最小宽度属性 min-width
