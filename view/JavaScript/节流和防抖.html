# 节流 - throttle

节流：在规定时间内，只让函数触发的第一次生效，后面触发的函数不生效

当频繁重复触发一个函数时，会对性能有很大压力，这时可以设置一个延时函数来减少事件函数的调用频率，达到性能提升的效果

应用：限制发送请求的时间间隔

实现：闭包，条件判断

闭包：防止条件判断的LastTime变量污染全局命名空间，也防止每次调用初始化LastTime

条件判断：利用Date.now()计算与上次执行函数时的时间戳的时间差，中间调用函数将无法执行，直到时间差大于delay才能再次执行函数，此时更新lastTime

初始化的lastTime可以是（当前时间-delay）之前的时间戳，最方便的还是设置0

    function throttle(fn, delay){
        //初始化时间为0，保证第一次能触发
        let lastTime = 0;
        //利用闭包保存lastTime
        //保证每次调用不会初始化lastTime
        return function(){
            let nowTime = Date.now();
            if(nowTime-lastTime > delay){
                //修正this指向
                fn.call(this);
                //执行后更新lastTime
                lastTime = nowTime;
            }
        }
    }


# 防抖 - debounce

防抖：让一个频繁触发的函数，在规定时间内只让最后一次生效，前面的不生效

当短时间内频繁触发函数，我们只关注最后一次的结果时，可以使用防抖来节约性能

应用：输入验证,限制请求

实现：闭包，延时函数setTimeout

闭包：防止延时函数的timer变量污染全局命名空间，也防止每次调用初始化timer

延时函数：当固定时间内再次调用函数，则会覆盖timer,导致上一个函数没有执行

    function debounce(fn, delay){
        let timer = null;
        //利用闭包保存timer
        //保证每次调用不会初始化lastTime
        return function(){
            if(timer){
                clearTimeout(timer);
            }
            //更新timer
            timer = setTimeout(()=>{
                fn.call(this);
            },delay);
        }
    }
