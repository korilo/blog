# # Git基础操作

  • git init
    • 作用：
      • 初始化一个仓库
  • git status
    • 作用：
      • 查看库状态
  • git add
    • 作用
      • 将指定文件加入到 暂存区 里
    • 用法：
      • git add file：加入指定文件
      • git add .：加入当前目录所有文件
  • git commit -m
    • 作用;
      • 将暂存区文件提交到 仓库
      • 每次提交一个版本，可以用git log查看提交了多少个版本
    • 用法：
      • git commit -m '提交的备注'
  • git log
    • 作用：
      • 查看所有的提交版本
      • 可以查看到每个版本的ID
      • 通过git checkout 版本号 可以回退到指定的版本
  • git checkout
    • 作用：
      • 回退指定版本
      • 从仓库内获取提交的版本并替换到 工作区 中
    • 用法：
      • git checkout 版本号，版本号可以通过git log获取
      • git checkout -- .：回退到上一个提交的版本

# # Git进阶操作

  • git checkout -b
    • 作用：
      • 复制一份原来的东西，开一个新分支
    • 用法：
      • git checkout -b 分支名
  • git branch
    • 作用：
      • 查看所有的分支
  • git checkout 分支名
    • 作用
      • 切换已有的分支
    • 用法：
      • git checkout master ：切换回主分支
  • git merge
    • 作用：
      • 合并当前分支与指定分支
    • 用法：
      • git merge 指定分支 -m '描述/备注'

# # 项目托管和协作

  • 托管：

    • 将项目放到服务器上以便于其他人员能够使用

  • 协作：

    • 针对项目，不同的人进行操作

  • gitHub和gitLab

    • gitHub:
      • 私密库收费、开源
    • gitLab:
      • 私密库免费、操作简单

  • gitHub使用：

    • 新建仓库
    • 与远端建立联系
      • git clone 远端地址
        • 克隆远端仓库到本地作为本地库，并保存与远端的联系
      • git remote add origin 远端地址
        • 在本地库直接与远端建立联系，添加远端库
    • 进行开发
    • 协作：
      • 本人操作：
        • 建立远端联系
        • git push -u origin master
          • 第一次提交
        • git push
          • 以后提交
      • 他人的操作
        • git clone
          • 克隆master分支至本地库
        • git clone -b 分支名
          • 克隆分支至本地库
        • git pull
          • 更新本地库与远端库保持一致
