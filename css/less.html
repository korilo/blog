    # 1. less基础

    less是为了实现css代码的层级关系，使css代码的结构与html一样，有层级关系

    • vscode插件

      • easy less
      • 保存时将less文件编译成css文件
      • 自动在当前目录生成同名的css文件

    • less注释

      • //：不会被编译到css文件中去
      • /**/：会被编译到css文件中

    • less嵌套规则

      • 普通嵌套：编译出来带空格，相当于后代选择器
      • &的使用：编译出来不带空格，相当于并集选择器

    • less变量@

      • 作用：

        • 对选择器或属性值用变量进行命名，并传入
        • 变量是块级作用域
        • 不常用，还是推荐直接写具体值

      • 属性值插值

        @myUrl: "http://blog.csdn.net/lee_magnum";
.meng {
	background-image: url("@{myUrl}/images/bg.png");
}

      • 选择器插值

        @Myname: meng1314;
.@{Myname} {
    width:1000000000000px;
}

      • 媒体查询插值

        • “~”后面的值是不被编译的，所以可以用作media query的参数
        • 该特性还可以用于避免less编译（计算）
        @singleQuery: ~"(max-width: 480px)";
@media screen, @singleQuery {
    div {
	width:2000px;
    }
}

      • 延时加载

        • 当css属性使用了变量，该css样式将在该作用域所有css变量声明后再调用
        • 类似于JavaScript的变量提升
        @width : 1px;

.meng {
	@width : 2px;
	.long {
		@width : 9000px;
		width:@width;
		@width : 22px;
	}
	width: @width;
}

    • less混合（mix-in）

      • 作用：类似于JavaScript函数封装，将重复的css样式封装，并可以被重复使用

        .center{
    position: absolute;
    top: 0;
    left: 0;
    right: 0;
    buttom: 0;
    margin: auto;
}

.item {
    .center;
}

      • 混合规则：

        • 普通混合（不带括号和参数）：

          • 会将封装的混合样式也编译到css文件里

        • 不带输出的混合（不带参数）：

          • 不会输出封装的混合样式到css文件里
          • 方法：在封装的样式名后加小括号，不传参数：.center(){}

        • 带参数的混合（带参数，形参）：

          • 可以对封装的属性进行传参
          • 方法：括号里穿参数，要加@符号：@形参
          .squareBox(@color, @size){
    height: @size;
    width: @size;
    background-color: @color;
}
.box{
    .squareBox(pink, 100px);
}

        • 参数带默认值的混合

          • 当参数传递数量与形参数不同时，将会报错
          • 加入默认值可以解决问题，没有传递的参数将采用默认值
          • 方法，在形参后加冒号定义默认值
          .squareBox(@color:red, @size:100px){}

        • 指定形参传递（指定传参）

          • 当只想传其中的一个参数，但又不是第一个形参所指定的参数时，可以指定该参数传递的形参
          • 方法：用@进行指定@size:200px
          .box {
    .squareBox(@size: 200px);
}

        • 匹配模式（带标识符）

          • 当要修改多属性值的属性（如border/margin....），每个属性值的顺序不确定时，可以通过定义多个状态，并用标识符来指定修改属性
          • 效果其实和定义多个封装一样，只是当里面重复样式多时可以少写点代码，等于对封装的样式再封装
          • 方法：
            • 将样式分成 **独立样式 **和 共同样式
            • 封装的 共同样式 样式指定形参是@_
            • 封装的 独立样式 指定 第一个形参 是标识符（自定义）
            • 引用样式时会自动引用共同样式以及指定的独立样式
          • 注意：
            • 共同样式里的形参必须要和独立样式里的一样，除了标识符，否则将不会传入共同样式
          .border(@_, @color){
    width: 100px;
    height: 100px;
    background-color: white;
}
.border(L, @color){
    border-style: dashed;
    border-width: 1px;
    border-color: transparent transparent transparent @color;
}
.border(T, @color){
    border-style: dashed;
    border-width: 1px;
    border-color: @color transparent transparent transparent;
}
.border(R, @color){
    border-style: dashed;
    border-width: 1px;
    border-color: transparent @color transparent transparent;
}
.border(B, @color){
    border-style: dashed;
    border-width: 1px;
    border-color: transparent transparent @color transparent;
}

.box {
    .border(L, red);
}

        • argumrnts变量

          • 当属性值是多值（如border），不想一个一个写变量，则可以直接用arguments
          .border(@w, @s, @c){
    border: @arguments;
}

.box{
    .border(1px, solid, pink);
}

    • less运算

      • 原生css也是可以进行运算的（使用calc( )方法），但会影响性能
      • less可以直接计算，当中只需一方带单位
      • 两个都有单位且不同时，以第一个为准

    • 编译出来的css属性值为计算的最终值

      width: (100 + 100px);

      
* 避免编译：
  
    * **“~”后面的值是不被编译的**
    * 可以使用~来避免编译过程的运算，让浏览器来进行运算
  * 注意两个都要带单位，因为不是less编译
  
    ```css
    width: ~"(100px + 100px)"

    ​

    • less继承

      • :extend()方法进行拼接
      • 作用：将共同的css属性封装在一个类中，再用并集选择器进行连接
      • 优缺点：性能比混合好，但是不灵活，无法传递参数
      • 注意：优先级问题，如果原来的类上有封装的属性，会进行覆盖
      .center{
    position: absolute;
    left: 0;
    top: 0;
    right: 0;
    buttom: 0;
    margin: auto;
}

box{
    &:extend(.center);
}
//也可以写成： box:extend(.center){ }

//css编译结果：
.center,box{
    position: absolute;
    left: 0;
    top: 0;
    right: 0;
    buttom: 0;
    margin: auto;
}
