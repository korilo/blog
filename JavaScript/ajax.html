          # 1. Ajax

          # 1.1 Ajax概念

          • 允许浏览器与服务器通信而无需刷新当前页面的技术都叫做Ajax

          • 实现：

            • flash
            • Java applet
            • 框架，修改页面的其中一个框架
            • 隐藏的iframe
            • XMLHtppRequest对象，JavaScript的一个扩展

          # 1.2 Ajax交互流程

          • 流程：

            • 创建Ajax对象
            • 建立通信
            • 发送请求
            • 接收数据

          • 创建Ajax对象

            • 兼容问题：chrome和IE、火狐的Ajax对象创建方式不同
          if(window.XMLHttpRequest){
    //chrome的Ajax对象
    var oAjax = new XMLHttpRequest();
}else{
    //火狐和IE6的Ajax对象
    var oAjax = new ActiveXOject('Microsoft.XMLHTTP');
}
          • 建立连接（通信）
            • oAjax.open('请求方式','地址',是否异步);
              • 请求方式：'GET' / 'POST'
              • 地址：文件路径
              • 异步：默认true；Ajax是JavaScript异步通信实现的
          oAjax.open('GET','1.text',true);

          • 发送请求

            • oAjax.send();

          • 接收数据

            • onreadystate响应函数：当通信状态发生改变时触发事件
            • readyState：通信状态码
              • Ajax对象的属性，用来记录当前的通信状态
                • 0：初始化，完成XMLHttpRequest对象的初始
                • 1：载入完成，开始发送请求
                • 2：发送请求完成
                • 3：开始解析，读取服务器的响应
                • 4：完成，对象读取服务器相应结束
              • 通信完成并不意味数据交互成功，所以还需要判断http的状态
            • status：http状态码
              • Ajax对象的属性，用来记录当前的http状态
                • 1xx：消息响应（少见）
                • 2xx：成功（200）
                • 3xx：重定向（304：not modified访问了缓存，数据未修改）
                • 4xx：客户端错误（404：客户端错误，没找到文件）
                • 5xx：服务器错误（
                  • 500：服务器内部错误，很大概率后台代码写错）
                  • 503：服务器问题
              • 总结：
                • 200||304：通信成功
            • responseText
              • Ajax的属性，保存服务器返回的文本
          oAjax.onreadystatechange = function(){
    //判断通信状态;  4：通信结束
    if(oAjax.readyState == 4){
        //判断http状态;  200,304:传输成功
        if(oAjax.status == 200|| oAjax.status == 304){
            //服务器返回文本
            alert(oAjax.responseText);
        }
    }
}

          # 1.3 Ajax库封装

          • Ajax库
          //ajax.js
function ajax(url ,fnSucc ,fnErr){
    
    // 1. 创建Ajax对象
    if(window.XMLHttpRequest){
        //chrome的Ajax对象
        var oAjax = new XMLHttpRequest();
    }else{
        //火狐和IE6的Ajax对象
        var oAjax = new ActiveXOject('Microsoft.XMLHTTP');
    };
    
    // 2. 建立通信
    oAjax.open('GET', url+ '?t='+Math.random() ,true);
    
    // 3. 发送请求
    oAjax.send();
    
    // 4. 获取数据
    oAjax.onreadystatechange = function(){
        //判断通信状态;  4：通信结束
        if(oAjax.readyState == 4){
            //判断http状态;  200,304:传输成功
            if(oAjax.status == 200|| oAjax.status == 304){
                //成功时回调函数
                fnSucc(oAjax.responseText);
            }else{
                //失败时回调函数
                fnErr(oAjax.status);
            }
        };
    };
    
}

          # 1.4 Ajax的使用

          • ajax用法

            ajax(请求路径，成功时的回调函数，失败时的回调函数);

            • 请求路径：即想要访问的服务器资源路径
              • 缓存问题：请求路径 相同时 优先访问缓存内的数据而不是服务器端的数据，造成数据更新不及时，不同步
              • 解决：在路径后加查询码'?t='+Math.random()
                • t可以是任何字符或字符串
                • 拼接个随机数可以确保每次的查询码不一样
                • 可以在ajax.js里补充
            • 成功时的回调函数
              • 参数：
                • data：获取的服务器返回数据
                • 需要
            • 失败时的回调函数
              • 参数：
                • err：errorCode，失败的http状态码
          //引用ajax.js
<script scr="ajax.js"> </script>
<script>
	//使用ajax发送请求
	ajax(url, function(){}, function(){});
</script>
          • 注意：
            • 缓存问题：用查询码随机数解决
            • 从服务器返回回来的都是字符串/JSON字符串，需要用JSON.parse(data)进行解析
              • JSON.parse()兼容问题：IE8上不支持
              • 解决：加上<meta http-equiv="X-UA-Compatible" content="ie=edge">，且必须加上HTML5的文档声明
            • 乱码问题： 文件与浏览器编码方式不同，一般统一 UTF-8
            • ajax请求服务器的文件跟后缀名无关，只读取出实际内容

          # 2. html表单

          # 2.1 表单提交三要素

          • action
            • form标签属性，提交地址
            • 不加http://将访问本地，所以要输全网址
          • method
            • form标签属性，提交方式
            • GET(默认）：
              • 明文，不安全，url能看见，32k，有缓存
              • 适用于分享的场景
            • POST：
              • 密文，相对安全，url看不到，1G，没有缓存
              • 适用于大量数据以及隐私数据的提交
          • name
            • 表单标签属性，提交字段名
          <form action="http://www.baidu,com/" method="POST">
    <input type="text" name="user" id='' placehoder="请输入用户名" />
    <input type="password" name="pwd" id='' placehoder="请输入密码" />
    <input type="submit" value="提交" />
</form>

          • form表单提交特点：

            • 刷新整个页面，用户体验不好，浪费贷款==带宽
            • 不能从服务器拿回数据

          • 常见错误：

            • 返回的JSON数据格式错误
            • 中文乱码问题：
              • encodeURIComponent（数据）; ： 将中文转成编码，交给后台
              • decodeURIComponent（数据）; ： 将编码转成中文，一般后台使用
